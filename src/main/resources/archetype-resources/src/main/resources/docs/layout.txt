= ディレクトリ構造・主要ファイルの説明

kdappアーキタイプを使ったアプリケーションでは、以下の位置にファイルを配置する必要があります。

`src/main/java`
: Javaソースコードはこのディレクトリ以下に配置してください。
: 必ずパッケージで空間の切り分けを伴うようにしましょう。
: アーキタイプから生成する際に基準のパッケージ空間は指定しているはずなので、その下にJavaソースは配置してください。

`src/main/resources`
: プログラムから利用するデータ類を配置してください。
: プロパティファイルはファイル名末尾を".properties"にしておけば、配布パッケージ生成時にconfigディレクトリに配置されます。
: 利用者向けドキュメントは、docsディレクトリ以下に".txt"のファイル名末尾で設定しておけば、トップディレクトリに配置されます。
: このファイルはこの方法でアーカイブのトップに配置するようにしています。
: このディレクトリにあるdescriptor-bin.xmlはパッケージ化する際に利用するコンフィギュレーションです。訳もわからない状態でいじるのは止めておきましょう。

`src/test`以下
: テストコードをおくためのところです。今回はほとんど使っていません。
: このディレクトリの利用方法は、3年生選択「ソフトウェア開発実験」にて行います。

`pom.xml`
: このディレクトリを含むアーティファクトの制御ファイルです。Apache Mavenを手動で入れていれば、コマンドラインでもこのアーティファクトは操作可能です。
: このファイルも訳もわからない状態でいじるのは止めておきましょう。
: Eclipse上でこのファイルを開けば、簡易画面にてアーティファクト名やバージョンなどは操作できるので、その程度にとどめておきましょう。
